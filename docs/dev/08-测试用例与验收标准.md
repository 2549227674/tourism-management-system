# 08-测试用例与验收标准

## 1. 测试目标

测试目标是验证旅游管理系统是否满足需求规格说明书中的功能要求、权限要求、数据一致性要求和基本性能要求，确保最终提交的源程序可运行、可演示、可验收。

## 2. 测试范围

| 范围 | 是否测试 | 说明 |
|---|---:|---|
| 用户注册登录 | 是 | 核心基础功能。 |
| 权限控制 | 是 | 普通用户不能访问管理员功能。 |
| 景点管理 | 是 | 管理员 CRUD 和用户浏览。 |
| 线路管理 | 是 | 管理员 CRUD、状态和名额。 |
| 订单预订 | 是 | 用户预订、取消、管理员处理。 |
| 评论审核 | 是 | 评论待审核、通过、驳回。 |
| 收藏管理 | 是 | 添加、重复收藏、取消。 |
| 公告资讯 | 是 | 发布、下架、查看。 |
| 数据统计 | 是 | 数量和金额统计。 |
| 在线支付 | 否 | 不在项目范围内。 |

## 3. 测试环境

| 项目 | 配置 |
|---|---|
| 操作系统 | Windows 10/11 或其他开发环境。 |
| 后端 | JDK 17 + Spring Boot。 |
| 前端 | Node.js 18 + Vue 3。 |
| 数据库 | MySQL 8.0。 |
| 浏览器 | Chrome、Edge 或 Firefox。 |

## 6. 验收标准

### 6.1 必须通过

1. 源程序可按运行指南启动；
2. 管理员可以登录后台；
3. 普通用户可以注册登录；
4. 景点和线路可管理、可浏览；
5. 用户能预订线路并生成订单；
6. 管理员能处理订单；
7. 权限控制有效；
8. 数据库脚本可执行；
9. 交付文档完整。

### 6.2 建议通过

1. 评论审核流程完整；
2. 收藏功能可用；
3. 公告发布和下架可用；
4. 后台统计数据正确；
5. 项目总结报告包含真实测试结果。
