# 01-技术栈与开发约束

## 1. 统一技术栈

| 层次 | 技术 | 版本建议 | 说明 |
|---|---|---|---|
| 后端语言 | Java | 17 | 统一使用 JDK 17。 |
| 后端框架 | Spring Boot | 3.x | 构建 REST API。 |
| 安全认证 | Spring Security + JWT | 与 Spring Boot 兼容 | 管理登录态和权限。 |
| ORM | MyBatis-Plus | 3.5.x | 简化 CRUD。 |
| 数据库 | MySQL | 8.0 | 统一使用 utf8mb4。 |
| 前端框架 | Vue | 3.x | 单页应用。 |
| 构建工具 | Vite | 5.x | 前端开发服务器。 |
| UI 组件 | Element Plus | 2.x | 表格、表单、弹窗。 |
| HTTP | Axios | 1.x | 前端请求后端。 |
| 状态管理 | Pinia | 2.x | 保存用户信息和 token。 |
| 版本管理 | Git | 2.x | 小组协作。 |
| 文档 | Markdown | - | 所有文档统一 Markdown。 |

## 2. 架构约束

1. 采用单体后端 + 单体前端，不拆分微服务；
2. 后端仅提供 REST API，不在后端渲染页面；
3. 前端通过 Axios 调用后端接口；
4. 数据库只使用 MySQL，不同时引入 PostgreSQL、MongoDB 等；
5. 初版不引入 Redis、消息队列、ElasticSearch 等额外中间件；
6. 图片先使用 URL 或本地静态路径，不强制实现文件上传服务。

## 3. 编码规范

### 3.1 后端命名

| 类型 | 命名示例 |
|---|---|
| Controller | `RouteController`、`AdminRouteController` |
| Service | `RouteService` |
| Service 实现 | `RouteServiceImpl` |
| Mapper | `RouteMapper` |
| Entity | `TourRoute` |
| DTO | `RouteCreateDTO` |
| VO | `RouteDetailVO` |

### 3.2 前端命名

| 类型 | 命名示例 |
|---|---|
| 页面文件 | `RouteList.vue`、`AdminOrderList.vue` |
| API 文件 | `routeApi.ts`、`orderApi.ts` |
| Store | `userStore.ts` |
| 路由名称 | `RouteList`、`AdminDashboard` |

### 3.3 数据库命名

1. 表名使用小写下划线：`tour_route`；
2. 字段名使用小写下划线：`created_at`；
3. 主键统一为 `id`；
4. 创建和更新时间统一为 `created_at`、`updated_at`；
5. 逻辑状态字段统一使用 `status`。

## 4. 接口约束

### 4.1 路径规范

| 类型 | 示例 |
|---|---|
| 公开接口 | `/api/spots`、`/api/routes` |
| 用户接口 | `/api/orders/my`、`/api/favorites` |
| 管理员接口 | `/api/admin/spots`、`/api/admin/orders` |

### 4.2 响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

### 4.3 分页格式

请求参数：

```text
page=1&pageSize=10&keyword=xxx
```

响应数据：

```json
{
  "records": [],
  "total": 0,
  "page": 1,
  "pageSize": 10
}
```

## 5. 禁止事项

1. 不得在中途随意更换技术栈；
2. 不得绕过接口文档直接改字段；
3. 不得将数据库密码、真实密钥提交到仓库；
4. 不得只做前端静态页面而无后端数据；
5. 不得只靠前端隐藏按钮实现权限控制；
6. 不得新增超出课程周期的大型功能。

## 6. 变更规则

技术变更必须满足以下条件：

1. 说明变更原因；
2. 评估对已有代码和文档的影响；
3. 全体成员同意；
4. 同步修改 README、CLAUDE.md 和相关开发文档；
5. 确认不会影响最终交付时间。
