# 04-后端接口设计说明

## 1. 基本约定

### 1.1 基础地址

本地开发环境：

```text
http://localhost:8080/api
```

### 1.2 请求格式

- GET 请求使用 query 参数；
- POST/PUT 请求使用 JSON body；
- 登录后请求需要在 Header 中携带 token。

```text
Authorization: Bearer <token>
```

### 1.3 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 1.4 常用错误码

| code | 说明 |
|---:|---|
| 0 | 成功。 |
| 400 | 请求参数错误。 |
| 401 | 未登录或登录已过期。 |
| 403 | 无权限。 |
| 404 | 数据不存在。 |
| 409 | 业务冲突，例如重复收藏、名额不足。 |
| 500 | 系统内部错误。 |

## 2. 认证接口

### 2.1 用户注册

```text
POST /auth/register
```

请求：

```json
{
  "username": "user01",
  "password": "user123",
  "confirmPassword": "user123",
  "phone": "13800000000",
  "email": "user01@example.com"
}
```

响应：

```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "user01"
  }
}
```

### 2.2 用户登录

```text
POST /auth/login
```

请求：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token-example",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "ADMIN"

  }

```

## 6. 线路接口

### 6.1 公开查询

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/routes` | 公开 | 线路分页列表（仅 OPEN、FULL 状态）。 |
| GET | `/routes/{id}` | 公开 | 线路详情（仅 OPEN、FULL 状态，DRAFT/CLOSED 返回 404）。 |

查询参数：

| 参数 | 说明 |
|---|---|
| page | 页码，默认 1。 |
| pageSize | 每页条数，默认 10。 |
| keyword | 按名称或行程安排模糊查询。 |
| status | 按状态筛选，只允许 OPEN 或 FULL，其他值返回 400。 |

### 6.2 管理接口

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/admin/routes` | ADMIN | 线路分页查询（含全部状态）。 |
| GET | `/admin/routes/{id}` | ADMIN | 线路详情。 |
| POST | `/admin/routes` | ADMIN | 新增线路。 |
| PUT | `/admin/routes/{id}` | ADMIN | 修改线路。 |
| DELETE | `/admin/routes/{id}` | ADMIN | 删除线路（有关联订单时返回 409）。 |
| PUT | `/admin/routes/{id}/status` | ADMIN | 修改线路状态（参数 status=DRAFT/OPEN/FULL/CLOSED）。 |

新增线路请求：

```json
{
  "name": "青山湖一日游",
  "itinerary": "08:00 集合出发；10:00 游览青山湖；15:00 返回。",
  "price": 199.00,
  "departureTime": "2026-07-01T08:00:00",
  "quota": 30,
  "status": "OPEN",
  "coverImageUrl": "/images/route-qingshan.jpg"
}
```

字段校验规则：

| 字段 | 规则 |
|---|---|
| name | 必填。 |
| itinerary | 必填。 |
| price | 必填，不能小于 0。 |
| departureTime | 必填。 |
| quota | 必填，必须大于 0。 |
| bookedCount | 选填，不传默认 0；不能小于 0，不能大于 quota。 |
| status | 选填，不传默认 DRAFT；只允许 DRAFT/OPEN/FULL/CLOSED。 |

状态说明：

| 状态 | 说明 |
|---|---|
| DRAFT | 草稿，不在公开端展示。 |
| OPEN | 开放预订，公开端可见。 |
| FULL | 名额已满，公开端可见。 |
| CLOSED | 已关闭，不在公开端展示。 |

删除约束：如果线路上已有订单，禁止删除，返回 409："该线路已有订单，不能删除，可改为关闭状态"。

## 7. 订单接口

### 7.1 用户订单

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| POST | `/orders` | USER/ADMIN | 创建订单。 |
| GET | `/orders/my` | USER/ADMIN | 我的订单列表（仅自己的订单）。 |
| GET | `/orders/{id}` | USER/ADMIN | 我的订单详情（仅自己的订单，否则 404）。 |
| PUT | `/orders/{id}/cancel` | USER/ADMIN | 取消订单（仅自己的订单）。 |

创建订单请求：

```json
{
  "routeId": 1,
  "peopleCount": 2,
  "contactName": "张三",
  "contactPhone": "13800000000"
}
```

创建订单响应：

```json
{
  "code": 0,
  "message": "预订成功，等待管理员处理",
  "data": {
    "orderNo": "TR202607010800123456",
    "totalAmount": 398.00,
    "status": "PENDING"
  }
}
```

查询参数（`/orders/my`）：

| 参数 | 说明 |
|---|---|
| page | 页码，默认 1。 |
| pageSize | 每页条数，默认 10。 |
| status | 按状态筛选，可选。 |

创建订单校验规则：

| 字段 | 规则 |
|---|---|
| routeId | 必填，线路必须存在且状态为 OPEN。 |
| peopleCount | 必填，必须大于 0，不能超过剩余名额。 |
| contactName | 必填。 |
| contactPhone | 必填。 |

取消订单规则：只能取消自己的订单；PENDING 和 CONFIRMED 可取消；CANCELLED、REJECTED、COMPLETED 不可取消。取消后释放名额。

### 7.2 管理员订单

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/admin/orders` | ADMIN | 订单分页查询。 |
| GET | `/admin/orders/{id}` | ADMIN | 订单详情。 |
| PUT | `/admin/orders/{id}/confirm` | ADMIN | 确认订单（仅 PENDING）。 |
| PUT | `/admin/orders/{id}/reject` | ADMIN | 驳回订单（仅 PENDING，释放名额）。 |
| PUT | `/admin/orders/{id}/complete` | ADMIN | 完成订单（仅 CONFIRMED）。 |
| PUT | `/admin/orders/{id}/cancel` | ADMIN | 管理员取消订单（PENDING/CONFIRMED，释放名额）。 |

查询参数（`/admin/orders`）：

| 参数 | 说明 |
|---|---|
| page | 页码，默认 1。 |
| pageSize | 每页条数，默认 10。 |
| keyword | 按订单号、联系人、联系电话模糊查询。 |
| status | 按状态筛选，可选。 |

驳回/取消订单请求（均可选）：

```json
{
  "remark": "该线路临时调整，暂不接受预订"
}
```

订单状态流转：

| 当前状态 | 可执行操作 | 目标状态 | 是否释放名额 |
|---|---|---|---|
| PENDING | 管理员确认 | CONFIRMED | 否 |
| PENDING | 管理员驳回 | REJECTED | 是 |
| PENDING | 用户/管理员取消 | CANCELLED | 是 |
| CONFIRMED | 管理员完成 | COMPLETED | 否 |
| CONFIRMED | 用户/管理员取消 | CANCELLED | 是 |
| REJECTED | — | 终态 | 已释放 |
| CANCELLED | — | 终态 | 已释放 |
| COMPLETED | — | 终态 | 不释放 |

订单金额 = 线路价格 × 预订人数，由后端计算，不接受前端传入。

## 12. 接口联调顺序

1. 登录注册接口；
2. 景点和线路公开查询；
3. 管理员景点和线路维护；
4. 用户创建订单；
5. 管理员处理订单；
6. 评论收藏；
7. 公告和统计。
