# 04-后端接口设计说明

## 1. 基本约定

### 1.1 基础地址

本地开发环境：

```text
http://localhost:8080/api
```

### 1.2 请求格式

- GET 请求使用 query 参数；
- POST/PUT 请求使用 JSON body；
- 登录后请求需要在 Header 中携带 token。

```text
Authorization: Bearer <token>
```

### 1.3 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 1.4 常用错误码

| code | 说明 |
|---:|---|
| 0 | 成功。 |
| 400 | 请求参数错误。 |
| 401 | 未登录或登录已过期。 |
| 403 | 无权限。 |
| 404 | 数据不存在。 |
| 409 | 业务冲突，例如重复收藏、名额不足。 |
| 500 | 系统内部错误。 |

## 2. 认证接口

### 2.1 用户注册

```text
POST /auth/register
```

请求：

```json
{
  "username": "user01",
  "password": "user123",
  "confirmPassword": "user123",
  "phone": "13800000000",
  "email": "user01@example.com"
}
```

响应：

```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "user01"
  }
}
```

### 2.2 用户登录

```text
POST /auth/login
```

请求：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token-example",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "ADMIN"
    }
  }
}
```

## 3. 用户接口

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/users/me` | USER/ADMIN | 获取当前用户信息。 |
| PUT | `/users/me` | USER/ADMIN | 修改当前用户资料。 |
| PUT | `/users/me/password` | USER/ADMIN | 修改密码。 |
| GET | `/admin/users` | ADMIN | 管理员查询用户列表，可选。 |

## 4. 景点分类接口

### 4.1 公开查询

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/categories` | 公开 | 查询已启用的分类列表。 |

### 4.2 管理接口

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/admin/categories` | ADMIN | 查询全部分类（含禁用）。 |
| POST | `/admin/categories` | ADMIN | 新增分类。 |
| PUT | `/admin/categories/{id}` | ADMIN | 修改分类。 |
| DELETE | `/admin/categories/{id}` | ADMIN | 删除分类。 |

新增分类请求：

```json
{
  "name": "自然风光",
  "sortOrder": 1
}
```

## 5. 景点接口

### 5.1 公开查询

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/spots` | 公开 | 景点分页列表（仅 ON 状态）。 |
| GET | `/spots/{id}` | 公开 | 景点详情。 |

查询参数：

| 参数 | 说明 |
|---|---|
| page | 页码，默认 1。 |
| pageSize | 每页条数，默认 10。 |
| keyword | 按名称或地址模糊查询。 |
| categoryId | 按分类 ID 筛选。 |

### 5.2 管理接口

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/admin/spots` | ADMIN | 景点分页查询（含 OFF 状态）。 |
| POST | `/admin/spots` | ADMIN | 新增景点。 |
| PUT | `/admin/spots/{id}` | ADMIN | 修改景点。 |
| DELETE | `/admin/spots/{id}` | ADMIN | 删除景点。 |
| PUT | `/admin/spots/{id}/status` | ADMIN | 上架或下架（参数 status=ON/OFF）。 |

新增景点请求：

```json
{
  "name": "青山湖风景区",
  "categoryId": 1,
  "address": "某市青山湖路 1 号",
  "ticketPrice": 60.00,
  "openTime": "08:00-18:00",
  "introduction": "以湖泊和山地景观为主的休闲旅游景区。",
  "imageUrl": "/images/spot-qingshan.jpg"
}
```

## 7. 订单接口

### 7.1 用户订单

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| POST | `/orders` | USER | 创建订单。 |
| GET | `/orders/my` | USER | 我的订单列表。 |
| GET | `/orders/{id}` | USER | 我的订单详情。 |
| PUT | `/orders/{id}/cancel` | USER | 取消订单。 |

创建订单请求：

```json
{
  "routeId": 1,
  "peopleCount": 2,
  "contactName": "张三",
  "contactPhone": "13800000000"
}
```

创建订单响应：

```json
{
  "code": 0,
  "message": "预订成功，等待管理员处理",
  "data": {
    "orderNo": "TR202607010001",
    "totalAmount": 398.00,
    "status": "PENDING"
  }
}
```

### 7.2 管理员订单

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/admin/orders` | ADMIN | 订单分页查询。 |
| GET | `/admin/orders/{id}` | ADMIN | 订单详情。 |
| PUT | `/admin/orders/{id}/confirm` | ADMIN | 确认订单。 |
| PUT | `/admin/orders/{id}/reject` | ADMIN | 驳回订单。 |
| PUT | `/admin/orders/{id}/complete` | ADMIN | 完成订单。 |
| PUT | `/admin/orders/{id}/cancel` | ADMIN | 管理员取消订单。 |

驳回订单请求：

```json
{
  "remark": "该线路临时调整，暂不接受预订"
}
```


## 12. 接口联调顺序

1. 登录注册接口；
2. 景点和线路公开查询；
3. 管理员景点和线路维护；
4. 用户创建订单；
5. 管理员处理订单；
6. 评论收藏；
7. 公告和统计。
