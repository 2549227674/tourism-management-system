# 09-Claude-Code执行安排表

## 1. 使用目标

Claude Code 用于辅助生成和修改旅游管理系统源代码、测试脚本和项目文档。使用时必须遵循本项目既定技术栈、功能范围、目录结构和文档同步规则。

## 2. 执行原则

1. 先读文档，再改代码；
2. 按模块开发，不跨模块大范围重构；
3. 每次只解决一个明确任务；
4. 不随意更换技术栈；
5. 修改代码后同步更新相关 Markdown 文档；
6. 保证最终交付文档与源代码一致。

## 3. 推荐执行顺序

| 顺序 | 任务 | 输入文档 | 输出 | 检查点 |
|---:|---|---|---|---|
| 1 | 创建项目骨架 | README、技术栈约束 | backend、frontend 基础目录 | 可启动。 |
| 2 | 建立数据库脚本 | 数据库设计说明 | schema.sql、data.sql | MySQL 可执行。 |
| 3 | 实现认证权限 | 接口设计、权限规则 | 登录注册接口、JWT | USER/ADMIN 区分。 |
| 4 | 实现景点模块 | 功能规划、接口设计 | 景点接口和页面 | CRUD 与前台浏览。 |
| 5 | 实现线路模块 | 功能规划、接口设计 | 线路接口和页面 | 状态和名额正确。 |
| 6 | 实现订单模块 | 业务规则、测试用例 | 订单创建/取消/处理 | 名额释放正确。 |
| 7 | 实现评论收藏 | 业务规则 | 评论审核、收藏 | 审核状态正确。 |
| 8 | 实现公告统计 | 接口设计 | 公告管理、统计卡片 | 管理员可见。 |
| 9 | 联调修复 | 测试用例 | 修复缺陷 | 用例通过。 |
| 10 | 文档同步 | 交付文档更新规则 | 更新全部文档 | 文档与代码一致。 |

## 4. Claude Code 任务模板

### 4.1 新增模块模板

```text
请阅读 README.md、CLAUDE.md、docs/dev/01-技术栈与开发约束.md、docs/dev/04-后端接口设计说明.md。
现在只实现【模块名称】模块，不要修改无关模块，不要更换技术栈。
要求：
1. 后端接口路径遵循文档；
2. 前端页面遵循 docs/dev/05-前端页面与交互说明.md；
3. 数据库字段遵循 docs/dev/03-数据库设计说明.md；
4. 完成后更新相关文档中的状态或接口说明；
5. 给出本次修改文件清单和运行/测试方法。
```

### 4.2 修复缺陷模板

```text
请根据以下缺陷修复代码：
缺陷编号：BUG-XXX
缺陷描述：……
复现步骤：……
期望结果：……
限制：只修改与该缺陷相关的代码，不进行大范围重构。
修复后请更新 docs/dev/08-测试用例与验收标准.md 中对应缺陷状态，并说明验证方法。
```

### 4.3 文档同步模板

```text
请检查当前源代码与 docs/deliverables、docs/dev 中的描述是否一致。
重点检查：技术栈、接口路径、数据库字段、业务状态、运行命令、默认账号。
只修改文档，不修改代码。输出修改清单。
```

## 5. 分模块执行安排

### 5.1 认证与权限

| 步骤 | 说明 |
|---|---|
| 读取文档 | 技术栈、接口设计、权限规则。 |
| 生成代码 | `AuthController`、`UserService`、JWT 工具、路由守卫。 |
| 验证 | 管理员登录进后台，普通用户不能进后台。 |
| 更新文档 | 接口设计、运行指南默认账号。 |

### 5.2 景点和线路

| 步骤 | 说明 |
|---|---|
| 后端 | Entity、Mapper、Service、Controller。 |
| 前端 | 列表、详情、后台 CRUD 页面。 |
| 验证 | 新增后前台可见，下架后前台不可见。 |
| 更新文档 | 数据库字段和接口字段保持一致。 |

### 5.3 订单

| 步骤 | 说明 |
|---|---|
| 后端 | 订单创建、取消、确认、驳回、完成。 |
| 前端 | 预订弹窗、我的订单、后台订单表。 |
| 验证 | 订单状态和线路名额变化。 |
| 更新文档 | 测试用例和业务规则。 |

## 6. 每次执行后的检查清单

| 检查项 | 是否必须 |
|---|---:|
| 项目能否编译或启动 | 是 |
| 是否新增或修改接口 | 是，需更新接口文档 |
| 是否修改数据库字段 | 是，需更新数据库文档和 SQL |
| 是否修改业务状态 | 是，需更新权限与业务规则文档 |
| 是否影响运行命令 | 是，需更新 README 和运行指南 |
| 是否有测试或验证步骤 | 是 |
| 是否列出修改文件 | 是 |

## 7. 禁止让 Claude Code 执行的事项

1. 不要把项目改成其他技术栈；
2. 不要删除 docs 目录；
3. 不要引入支付、地图、短信等新复杂模块；
4. 不要生成与旅游管理系统无关的示例业务；
5. 不要将密码明文写入公开配置；
6. 不要在没有说明的情况下大范围重命名接口和字段；
7. 不要只更新代码而不更新相关文档。
