# 00-项目总体说明

## 1. 项目定位

旅游管理系统是软件工程课程设计项目，目标是在有限周期内完成一个小型、可运行、文档完整的 Web 系统。系统围绕“旅游信息展示 + 旅游线路预订 + 后台管理”展开，避免设计过度复杂。

## 2. 项目目标

1. 完成用户端与管理员端的基础功能；
2. 输出可运行源程序和数据库脚本；
3. 输出课程要求的正式交付文档；
4. 保持需求、设计、接口、测试和源代码一致；
5. 体现 4 人左右小组协作开发过程。

## 3. 系统角色

| 角色 | 能力 |
|---|---|
| 游客 | 浏览公开景点、线路和公告。 |
| 普通用户 | 注册登录、预订线路、取消订单、收藏、评论、维护个人信息。 |
| 管理员 | 管理景点、线路、订单、评论、公告和统计数据。 |

## 4. 功能边界

### 4.1 必须完成

- 用户注册登录；
- 管理员登录和权限控制；
- 景点信息管理；
- 旅游线路管理；
- 订单预订、取消和后台处理；
- 数据统计；
- 源程序运行指南。

### 4.2 优先完成

- 评论提交和审核；
- 收藏景点或线路；
- 公告发布和查看。

### 4.3 不做或后续再做

- 在线支付；
- 实名认证；
- 地图定位；
- 复杂推荐算法；
- 多商家入驻；
- 微服务和分布式部署。

## 6. 开发流程

```mermaid
flowchart LR
    A[确定范围] --> B[需求分析]
    B --> C[数据库和接口设计]
    C --> D[前后端编码]
    D --> E[联调测试]
    E --> F[缺陷修复]
    F --> G[文档同步]
    G --> H[课程验收]
```

## 7. 质量要求

1. 代码能在本地按 README 运行；
2. 核心业务数据正确；
3. 管理员接口有权限保护；
4. 文档和代码模块名称一致；
5. 每位成员能说明自己负责部分，也能理解整体流程。
