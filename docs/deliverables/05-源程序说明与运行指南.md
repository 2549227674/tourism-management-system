> 文档状态：初版骨架 + 示例内容

## 1. 源程序组成

```text

├── backend/                  # Spring Boot 后端项目
```

## 3. 数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE tourism_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行脚本

在 MySQL 客户端中依次执行：

```bash
mysql -u tourism -p tourism_management < database/schema.sql
mysql -u tourism -p tourism_management < database/data.sql
```

### 3.3 配置数据库连接

后端配置文件位置：

```text
backend/src/main/resources/application.yml
```

示例配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/tourism_management?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: tourism
    password: tourism123
```

> 如本地 MySQL 账号不同，请修改 `username` 和 `password` 为实际值。

## 4. 后端运行方法

进入后端目录：

```bash
cd backend
mvn clean install
mvn spring-boot:run
```

默认后端访问地址：

```text
http://localhost:8080
```

接口前缀：

```text
/api
```

健康检查接口可设置为：

```text
GET http://localhost:8080/api/health
```

## 5. 前端运行方法

进入前端目录：

```bash
cd frontend
npm install
npm run dev
```

默认前端访问地址：

```text
http://localhost:5173
```

前端通过 Vite 代理将 `/api` 请求转发到后端 `http://localhost:8080`，配置位于 `frontend/vite.config.js`。如后端地址或端口不同，请修改该文件中的 proxy 配置。

## 7. 演示流程建议

### 7.1 用户端演示

1. 打开系统首页；
2. 查看景点列表和景点详情；
3. 查看线路列表和线路详情；
4. 使用普通用户登录；
5. 选择开放线路并提交预订订单；
6. 查看“我的订单”；
7. 收藏景点或线路；
8. 提交评论并观察评论状态。

### 7.2 管理员端演示

1. 使用管理员账号登录；
2. 进入后台首页查看统计数据；
3. 新增或修改一个景点；
4. 新增或修改一个旅游线路；
5. 查看用户提交的订单并确认；
6. 审核用户评论；
7. 发布一条公告；
8. 返回前台查看公告展示效果。

## 8. 常见问题

| 问题 | 可能原因 | 处理方法 |
|---|---|---|
| 后端启动失败 | 数据库未启动或密码错误 | 检查 MySQL 服务和 `application.yml`。 |
| 前端接口 404 | 接口前缀配置错误 | 检查 `VITE_API_BASE_URL`。 |
| 登录失败 | 初始化数据未导入或密码不一致 | 重新执行 `data.sql`。 |
| 中文乱码 | 数据库字符集不正确 | 使用 `utf8mb4` 字符集重建数据库。 |
| 普通用户能进后台 | 前端路由或后端权限未限制 | 检查路由守卫和后端鉴权。 |
